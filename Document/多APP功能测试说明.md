# 多APP功能测试说明

## 功能概述
现在SelectApp支持配置多个APP，格式为逗号分隔的数字（如：1,2,4）。系统会按照配置的顺序依次打开每个APP，并在每个APP中执行定位、粘贴、发送等后续流程。

## 测试步骤
1. 确保豆包、Kimi和Qwen应用已启动
2. 复制任意文本内容
3. 按下Win+F2打开自定义菜单
4. 选择"11. 多APP测试"菜单项
5. 观察系统是否依次在豆包、Kimi和Qwen中发送内容

## 预期结果
- 菜单项显示为"11. 多APP测试（豆包、Kimi、Qwen）"
- 系统会按照豆包→Kimi→Qwen的顺序依次发送内容
- 每个APP之间有500毫秒的延迟，确保切换完成

## 配置说明
在Config.ini文件中，MenuItem11的配置如下：
```
[MenuItem11]
Name=多APP测试
Content=这是一个测试多个APP功能的菜单项，将会在豆包、Kimi和Qwen中依次发送内容：{text}
NewChat=1
SendEnter=1
SelectApp=1,2,4
```

## 注意事项
- 确保所有目标APP已启动，否则会显示错误信息
- 如果某个APP配置不存在，会跳过该APP并继续处理下一个
- 单个APP的配置方式保持不变，确保向后兼容
